# Initial IT Venture Idea Log

## Idea 1: Student Project Management System

### Problem Area
University students often struggle to manage group projects, assignment deadlines, files, and task responsibilities. Project information is usually spread across chat groups, Google Drive, notebooks, and personal reminders.

### Target Users
University students who work on group assignments and semester projects.

### Current Alternative
Students currently use LINE groups, Google Drive, personal notes, and verbal reminders. However, these tools are not always organized in one place.

### Proposed IT Solution
A web-based Student Project Management System that helps students create project tasks, assign responsibilities, upload file links, track deadlines, and record weekly progress.

### Possible Technology
- HTML
- CSS
- JavaScript
- Local Storage
- GitHub
- Simple Dashboard

### Why This Is Suitable
This project is suitable because it can be developed as a simple web prototype within one semester. The team can test the idea with real university students and improve the system based on feedback.

## Idea 2: Student Attendance and Reminder System

### Problem Area
Students sometimes forget class schedules, attendance requirements, and important academic activities.

### Target Users
University students who need help managing class attendance and academic reminders.

### Current Alternative
Students use phone calendars, class group chats, or memory. These methods can be missed or ignored.

### Proposed IT Solution
A simple web-based attendance and reminder system where students can record classes, mark attendance, and receive reminders for upcoming classes or activities.

### Possible Technology
- HTML
- CSS
- JavaScript
- Local Storage
- Calendar Feature

### Why This Is Suitable
This idea can be built as a clickable or working prototype. It is easy to understand and can be validated with students.

## Idea 3: Campus Service Information System

### Problem Area
New students may not know where to find campus services such as registration, advising, food areas, transport, or student support offices.

### Target Users
New university students and visitors.

### Current Alternative
Students ask friends, check social media groups, or search manually around campus.

### Proposed IT Solution
A web-based campus service information system that provides service locations, opening hours, contact information, and simple guidance for students.

### Possible Technology
- HTML
- CSS
- JavaScript
- Search Function
- Map or Location Link

### Why This Is Suitable
This project is suitable for a semester MVP because the prototype can be created with simple web pages, search features, and campus information.
